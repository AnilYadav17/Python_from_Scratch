Q1: What are the key features of Python?
A: Python is known for its simplicity and readability, making it beginner-friendly. It is an interpreted, high-level, dynamically typed language with automatic memory management. It supports multiple programming paradigms (procedural, object-oriented, functional) and has a vast standard library. Python is also platform-independent.

Q2: Why is Python called an interpreted language?
A: Python is called interpreted because its code is executed line-by-line by an interpreter rather than being fully compiled into machine code before execution. This allows easier debugging and flexibility.

Q3: Difference between compiled and interpreted languages?
A: Compiled languages (like C) convert the entire program into machine code before execution, making them faster. Interpreted languages (like Python) execute code line-by-line, making them slower but easier to debug.

Q4: What makes Python dynamically typed?
A: In Python, variable types are determined at runtime. You don't need to declare a variable's type explicitly; Python assigns it automatically based on the value.

Q5: Why is Python platform independent?
A: Python code is compiled into bytecode, which is executed by the Python Virtual Machine (PVM). Since PVM exists on multiple platforms, the same code runs everywhere.

Q6: What is the advantage of Python being a high-level language?
A: High-level languages are closer to human language, making them easier to read, write, and maintain. They abstract away hardware-level details.

Q7: What are the limitations of Python?
A: Python is slower than compiled languages, consumes more memory, and is not ideal for mobile apps or high-performance computing tasks.

Q8: How does Python execute a program internally?
A: Python first converts source code into bytecode, then the Python Virtual Machine executes this bytecode.

Q9: What is bytecode in Python?
A: Bytecode is an intermediate, platform-independent representation of Python code generated after compilation.

Q10: What is PVM (Python Virtual Machine)?
A: PVM is the runtime engine that executes Python bytecode.

Q11: Difference between source code, bytecode, and machine code?
A: Source code is human-readable code. Bytecode is intermediate code. Machine code is binary instructions executed by the CPU.

Q12: Does Python compile code before execution?
A: Yes, Python compiles code into bytecode before executing it.

Q13: Where is bytecode stored?
A: Bytecode is stored in the __pycache__ directory as .pyc files.

Q14: Is Python fully interpreted?
A: No, Python is both compiled (to bytecode) and interpreted (executed by PVM).

Q15: What is compilation in Python?
A: Compilation is the process of converting source code into bytecode.

Q16: What is parsing in Python?
A: Parsing is analyzing the syntax of the code to ensure it follows language rules.

Q17: What is a parse tree?
A: A parse tree represents the syntactic structure of code based on grammar rules.

Q18: What is Abstract Syntax Tree (AST)?
A: AST is a simplified tree representation of source code used internally by Python.

Q19: Difference between parse tree and AST?
A: Parse tree shows full grammar structure; AST is simplified and more practical for execution.

Q20: What role does AST play in Python execution?
A: AST is used as an intermediate step before generating bytecode.

Q21: What is the compilation pipeline in Python?
A: Source code → Tokenization → Parsing → AST → Bytecode → Execution.

Q22: Steps from source code to execution?
A: Write code → Compile to bytecode → Execute via PVM.

Q23: What is lexical analysis?
A: Breaking code into tokens like keywords, identifiers, operators.

Q24: What is syntax analysis?
A: Checking whether tokens follow correct grammar rules.

Q25: What happens if there is a syntax error?
A: The program stops execution and shows an error message.

Q26: What is a code object?
A: A code object is the compiled bytecode representation of a block of Python code.

Q27: What is a .pyc file?
A: A .pyc file stores compiled bytecode for faster execution.

Q28: When are .pyc files created?
A: When a Python file is first executed or imported.

Q29: Purpose of __pycache__ folder?
A: To store compiled bytecode files.

Q30: Does Python recompile code every time?
A: No, it uses cached bytecode if the source hasn't changed.
